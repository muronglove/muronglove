  • problems

很久不用hexo了，今天突然想用一下，结果发现了很多问题，如下：

  • Error: Module version mismatch. Expected 48, got 47. hexo使用的时候会有这个提示，查了一下，应该是npm版本提升了之后不兼容的问题，尝试升级hexo和npm之后，也没能解决，不过并不影响使用
  • 使用hexo时没有进入相应的文件夹，hexo的使用必然要依靠文件夹，如果没有文件夹的话可以使用『hexo init blog』来创建一个blog文件夹，文件夹出现问题 的时候可以使用『hexo init』来恢复出厂设置，然后配置config.yml,其中最重要的是『deploy:
    type: git
    repository: git@github.com:muronglove/muronglove.git
    branch: gh-pages』部分，这部分设定了github的地址，还有一些页面的描述，不再赘述
  • 如果使用了git init，那么可能会把根目录传到github上，这个时候要参照条目2
  • 建立私钥和公钥，在写博客的时候就不需要重新输入密码了，网址：http://www.cnblogs.com/ayseeing/p/3572582.html
  • hexo的一些常用指令：hexo g：构建html文件；hexo d：部署文件；hexo s：本地服务器查看；hexo new “filename”：创建新文件
总结：温故而知新，知识要时常温习，在用到的时候才不会像新学的一样，以后要吸取这个教训！
